# C/C++ Dev Environment

## devenv.nix

devenv's `languages.c` module is minimal — add what you need explicitly.
Pin a specific clang version via versioned nixpkgs packages (`clang_19`, `clang-tools_19`, etc.).

```nix
{ pkgs, ... }:
let
  clang = pkgs.clang_19;
  clang-tools = pkgs.clang-tools_19;  # clangd, clang-format, clang-tidy
in
{
  languages.c = {
    enable = true;
    lsp.package = clang-tools;
  };

  packages = [
    clang
    clang-tools
    pkgs.cmake
    pkgs.ninja
    pkgs.bear       # generates compile_commands.json for Make projects
    pkgs.pkg-config
    pkgs.gdb
  ];

  env.CC = "${clang}/bin/clang";
  env.CXX = "${clang}/bin/clang++";
}
```

## compile_commands.json — the key to std:: navigation

clangd needs `compile_commands.json` to know where headers live (including Nix store paths).
Without it, `std::vector` go-to-definition will fail.

**CMake** — enable in `CMakeLists.txt`:
```cmake
set(CMAKE_EXPORT_COMPILE_COMMANDS ON)
```

Then build once and symlink so clangd finds it at the project root:
```bash
cmake -B build -G Ninja
ln -s build/compile_commands.json .
```

**Make** — use `bear` to generate it:
```bash
bear -- make
```

## .clangd

Project-level clangd config. The `Compiler` key ensures clangd resolves
the correct Nix store include paths when `compile_commands.json` is absent.

```yaml
CompileFlags:
  Compiler: clang++

InlayHints:
  Enabled: true
  ParameterNames: true
  DeducedTypes: true
```

## .clang-format

Matches your conform.nvim `--style=LLVM` default. Put this in the project
root to make it explicit and version-controlled.

```yaml
BasedOnStyle: LLVM
IndentWidth: 4
ColumnLimit: 100
```

## Neovim — mason.lua

Mason installs clangd and clang-format. They work with Nix projects as long
as `compile_commands.json` is present — the file embeds all Nix store paths.

```lua
tool_installer = {
  ensure_installed = {
    "clangd",
    "clang-format",
    -- ...
  },
}
```

## Neovim — lsp.lua

Your current clangd config — matches well for C++20 projects:

```lua
clangd = {
  cmd = {
    "clangd",
    "--background-index",
    "--clang-tidy",
  },
  capabilities = {
    offsetEncoding = "utf-8",  -- avoids encoding mismatch warnings
  },
  settings = {
    fallbackFlags = { "-std=c++20" },  -- used when compile_commands.json is absent
  },
},
```

`fallbackFlags` is the safety net for files not covered by `compile_commands.json`.
With the file present, clangd ignores it and uses the real flags.

## Neovim — conform.lua

```lua
formatters_by_ft = {
  c = { "clang-format" },
  cpp = { "clang-format" },
},
formatters = {
  clang_format = {
    prepend_args = { "--style=LLVM" },  -- overridden by .clang-format if present
  },
},
```

If a `.clang-format` file exists in the project root, clang-format uses it
and ignores `--style=LLVM`. The `prepend_args` is just the fallback.

```
├── .clangd

├── CMakeLists.txt

├── devenv.lock
├── .envrc

```

## Common commands

```bash
cmake -B build -G Ninja        # configure
cmake --build build            # build
bear -- make                   # generate compile_commands.json for Make
clang-tidy src/main.cpp        # lint
clang-format -i src/**/*.cpp   # format all files
```
